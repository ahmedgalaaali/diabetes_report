1. What is the distribution of time spent in the hospital?

CREATE VIEW

For this I created a “view” first, this step saves me some time in the next queries when I need to call the same query once again.

CREATE VIEW pt_condition AS 
    SELECT
        time_in_hospital AS time_spent,
        COUNT(*) AS frequency_of_duration,
        CASE 
            WHEN time_in_hospital > 7 THEN 'Acute Symptoms' ELSE 'Non-acute Symptoms'
            END AS patient_condition
    FROM diabetic_data
    GROUP BY time_in_hospital ORDER BY time_in_hospital;

If we take a look at the results, it shows that the periods patients stay in the hospital are in a range between 1 and 14 days, we categorized these periods according to the acuity of the patients as the patient who stays for a period longer than 7 days is considered a patient who has experienced acute symptoms.

2. Make a list of medical specialties with an average procedure count above 2.5 and total procedure count above 50

GROUP BY, HAVING

According to the analysis, this query shows the top medical specialties that match the criteria provided earlier. Here I didn’t use count function as it will count the rows for how many time this medical specialty got a procedure, so as long as num_procedures cells may have values greater than 1 we should calculate the total number of procedures per specialty.

data2 <- dbGetQuery(con,"SELECT
    medical_specialty,
    ROUND(AVG(num_procedures),2) AS avg_procedurs_count,
    SUM(num_procedures) AS total_procedurs_count
FROM diabetic_data
GROUP BY medical_specialty
HAVING
    AVG(num_procedures) > 2.5 AND
    SUM(num_procedures) > 50;")

4. Find the relationship between number of lab procedures and time spent in the hospital. Categorize the number of lab procedures into three categories (few, average and many).

Temporary table, STDDEV

The secret behind this analysis is calculating the standard deviation(s); which is how far the values are from the mean, the purpose behind this calculation is that we want to get the average range for the number of lab procedures.

Average Range = [(mean - s),(mean + s)]

So that I created a query with more details as follows:

head(dbGetQuery(con, "WITH avg_range AS(
    SELECT 
    ROUND(AVG(num_lab_procedures),0) - ROUND(STDDEV(num_lab_procedures),0) AS min_avg,
    ROUND(AVG(num_lab_procedures),0) + ROUND(STDDEV(num_lab_procedures),0) AS max_avg,
    MAX(num_lab_procedures) AS max_lab_procedures,
    MIN(num_lab_procedures) AS min_lab_procedures
FROM diabetic_data
)
SELECT
    CONCAT(min_lab_procedures, ' to ', (min_avg - 1)) AS few,
    CONCAT(min_avg, ' to ', max_avg) AS average,
    CONCAT((max_avg + 1), ' to ', max_lab_procedures) AS many
FROM avg_range;
"))
##       few  average      many
## 1 1 to 22 23 to 63 64 to 132

And finally got the desired results

head(dbGetQuery(con, "WITH avg_range AS(
    SELECT 
    ROUND(AVG(num_lab_procedures),0) - ROUND(STDDEV(num_lab_procedures),0) AS min_avg,
    ROUND(AVG(num_lab_procedures),0) + ROUND(STDDEV(num_lab_procedures),0) AS max_avg,
    MAX(num_lab_procedures) AS max_lab_procedures,
    MIN(num_lab_procedures) AS min_lab_procedures
FROM diabetic_data
),
rate AS(
SELECT
    time_in_hospital AS time_spent,
    num_lab_procedures,
    CASE
        WHEN num_lab_procedures BETWEEN min_lab_procedures AND  (avg_range.min_avg - 1)THEN 'Few'
        WHEN num_lab_procedures BETWEEN avg_range.min_avg AND avg_range.max_avg THEN 'Average'
        ELSE 'Many'
    END AS procedures_rate
FROM diabetic_data, avg_range
)
SELECT 
    procedures_rate,
    ROUND(AVG(time_spent), 2) AS avg_duration
FROM rate
GROUP BY procedures_rate ORDER BY avg_duration;"))
##   procedures_rate avg_duration
## 1             Few         3.22
## 2         Average         4.27
## 3            Many         6.25

5. A summary of the top 50 patients, their medications and lab procedures.

CONCAT, CASE WHEN

And for the final task, it was asked to show the top 50 patients with medications and number of lab procedures, I used CONCAT to make the summary in one cell and by identifying the gender using CASE I made each sentence match with the gender of each patient mentioned in the list.

The quert was as follows:

head(dbGetQuery(con, "SELECT
    CONCAT(
        (CASE WHEN gender = 'Male' THEN 'Mr. ' ELSE 'Mrs. ' END), 
        patient_nbr, ' ',
        (CASE WHEN readmitted = 'NO' THEN 'has not readmitted. ' ELSE 'has readmitted. ' END),
        (CASE WHEN gender = 'Male' THEN 'He had ' ELSE 'She had ' END),
        num_medications, ' medications and ', num_lab_procedures, ' lab procedures.'
    ) AS admission_summary
FROM diabetic_data
ORDER BY num_medications DESC, num_lab_procedures DESC
LIMIT 50;"))
